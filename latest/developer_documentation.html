Developer Documentation¶

This page is for people who are interested in how User Documentation ticks. It’s a pretty simple tool but there’s a couple places that deserve extra attention because there’s a bit complex or assume things about the user’s environment.

Broadly speaking, the most important parts of any Package / rez_sphinx set up are.

  • The generated conf.py file

  • The preprocess_entry_point module, which is used to make auto-help Auto-Append Help Tags work

  • Understanding the fact that User Documentation has no publishing capabilities by default and instead delegates all of that work to an optional dependency, rez_docbot.
