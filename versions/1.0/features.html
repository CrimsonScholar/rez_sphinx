Features

But above all else, rez_sphinx is thorough and well tested. Sphinx does an incredible job making documenting Python code easier but it just isn’t able to ensure documentation is kept up-to-date or link to existing documentation reliably.

rez_sphinx makes your documentation built to last. Here’s a summary of some of the features that rez_sphinx has to offer.

Base Features

  • Auto-initialize documentation

    • Supports combined or separate source / build folders

    • Extends sphinx-quickstart with optional, template files

    • Batch-publish documentation for source or installed Rez packages

      • CLI-based utilities to ensure “best results”

  • API documentation auto-generation / auto-syncing

  • Validate missing documentation, pre-build

  • Extremely configurable Configuring rez_sphinx and publisher settings

    • CLI-based configuration inspector - shows “unresolved” values

    • CLI-based “actual, resolved values” inspector

Intrinsic Features

  • Auto-link and cross reference documentation from multiple Rez packages at once

    • This works …

      • Even if package is external and you don’t want to change it.

        • Example: rez-pip installed packages.

      • Even if that Rez package isn’t build with rez_sphinx

  • Auto-discovers your documentation and integrates them into rez-help

  • Provides a default build_documentation rez-test command, for easier documentation building.

  • rez_sphinx provides sensible Sphinx conf.py values for your Sphinx projects

    • But you can still override the conf.py customizations with whatever you’d like

Opt-In Features

Opt-In Themes

You can set all Rez packages using rez_sphinx to use sphinx_rtd_theme (or whichever theme you prefer), using this Sphinx theme tutorial.

Opt-In Publishing

rez_sphinx can auto-publish your documentation.

It supports this using a release_hook.

As a publisher, rez_sphinx publish run provides these features:

  • One-to-many repositories-to-Rez package support - Multiple Rez packages in a single repository

  • One-to-one repository-to-Rez package support - One package per repository

  • “latest live” documentation which auto-updates with each new version

  • Versioned, locked documentation

    • Auto-linked and cross referenced documentation links against the versioned documentation. The result - links to external documentation don’t go out of date.
