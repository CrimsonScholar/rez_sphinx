rez_sphinx config

The command used to interact with rez_sphinx’s many configuration options. The config does nothing by itself but houses several sub-commands. See below for details.

rez_sphinx config check

A simple command which reads your currently-applied configuration settings. If everything is valid, the function returns. But if there’s unexpected keys or values, those details are shown in the terminal.

rez_sphinx config list-overrides

This commands shows all rez_sphinx configuration settings. The default command, rez_sphinx config list-overrides shows all overrides + default settings at once. If you want to see just the overrides and nothing else, choose rez_sphinx config list-overrides --sparse.

There’s also a --format option, for outputting to yaml.

rez_sphinx config list-defaults

Unlike rez_sphinx config list-overrides, which includes user-defined overrides, this command only considers the settings which rez_sphinx provides by default and ignores any user settings. This command good as reference or for setting up your own set of overrides.

As usual, --sparse and --format customize the output + display.

rez_sphinx config show

rez_sphinx config show is the main command to query configuration values for rez_sphinx. Don’t confuse this command with rez_sphinx view sphinx-conf because they do different things.

  • rez_sphinx config show: Get all rez_sphinx settings.

    • See Configuring rez_sphinx for example values

  • rez_sphinx view sphinx-conf

    • Queries your Sphinx conf.py for resolved values.

rez_sphinx config show returns values that, often times, are only suggested fallbacks. They aren’t guaranteed to make it into your documentation settings. rez_sphinx view sphinx-conf however is what Sphinx actually sees when it’s building your packages’s documentation and should be preferred for debugging a specific package.

rez_sphinx config show-all

This command shows every setting which rez_sphinx supports. “.” note nested dictionaries. For example, auto_help.filter_by would be written in a configuration as:

optionvars = {
    "rez_sphinx": {
        "auto_help": {
            "filter_by": "prefer_generated",
        },
    },
}

Each printed preference path from this command has an effect on rez_sphinx. See Configuring rez_sphinx to learn about those preference path options.

rez_sphinx config show build_documentation_key

This important command returns a rez-test string label. By default, “build_documentation”. This label, which may seem simple and unimportant, is the glue that most of rez_sphinx relies on for its automated processes.

Don’t change this configuration value unless you know what you’re doing. And if you must, change it at the How To Configure All Rez Packages level, affecting all packages. Do not set this at the Per-Package Configuration level.

Configuration Options

So as you can see, rez_sphinx config queries your active configuration and default values. But you may be wondering “how do I customize rez_sphinx, myself”? Head over to Configuring rez_sphinx to learn more.
